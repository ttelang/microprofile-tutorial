== Overview

This project demonstrates an e-commerce microservice. It provides REST endpoints for product management and showcases Jakarta EE 10 and MicroProfile 7.1 features in a practical application.

== Development Environment

This project includes a GitHub Codespace configuration with:

* Maven build tools
* VS Code extensions for Java and MicroProfile development

SDKMAN allows easy switching between Java versions:

[source,bash]
----
# List installed Java versions
sdk list java | grep -E ".*\s+installed"

# Switch to JDK 21 for the current shell session
sdk use java 21.0.9-ms

# Make JDK 21 the default for all sessions
sdk default java 21.0.9-ms

# Verify the current Java version

== REST API

This module implements a RESTful API for managing products using Jakarta EE and the REST architectural style. Each product is exposed as a resource with a unique URI. The API currently supports retrieval operations only:

* `GET /catalog/api/products` – returns a list of all products
* `GET /catalog/api/products/{id}` – returns a single product by its identifier

=== Creating a RESTful API

A RESTful API defines resources and the operations clients can perform on them. Here, products are the main resource and the API is designed around standard HTTP methods:

* `GET` to read product data
* `POST`, `PUT`, and `DELETE` will be added later to support full CRUD operations

The URI structure is based on resources, so `/catalog/api/products` represents the collection of all products, while `/catalog/api/products/{id}` represents a specific product.

=== Creating an Entity Class

The product entity class models the data for each product. In this example, `Product` is a plain Java object with fields such as `id`, `name`, `description`, and `price`.

Using wrapper types like `Long` and `Double` is important for JSON-B serialization because they can represent `null` values. The entity class also provides a no-argument constructor and getters/setters so the JSON binding provider can instantiate and populate objects automatically.

[source,java]
----
package io.microprofile.tutorial.store.product.entity;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class Product {
    private Long id;
    private String name;
    private String description;
    private Double price;
}
----

=== Creating a Service Class

A service class contains the business logic and data access for the resource. It holds the product data and provides methods to retrieve products. The service layer is managed by the container as an application-scoped bean, ensuring a single instance exists for the lifetime of the application.

[source,java]
----
package io.microprofile.tutorial.store.product.service;

import io.microprofile.tutorial.store.product.entity.Product;
import jakarta.enterprise.context.ApplicationScoped;
import java.util.*;

@ApplicationScoped
public class ProductService {
    private List<Product> products = new ArrayList<>();

    public ProductService() {
        products.add(new Product(1L, "Laptop", "High-performance laptop", 999.99));
        products.add(new Product(2L, "Mouse", "Wireless mouse", 29.99));
        products.add(new Product(3L, "Keyboard", "Mechanical keyboard", 149.99));
    }

    public List<Product> getProducts() {
        return products;
    }

    public Optional<Product> getProductById(Long id) {
        return products.stream()
                .filter(p -> p.getId().equals(id))
                .findFirst();
    }
}
----

=== Creating a Resource Class

The JAX-RS resource class maps HTTP requests to Java methods. `ProductResource` is responsible for handling incoming requests under the `/catalog/api/products` path and returning JSON responses.

The resource class uses annotations like `@Path`, `@GET`, and `@Produces(MediaType.APPLICATION_JSON)` to define endpoints and the media type that the endpoints produce. It injects the `ProductService` to retrieve data.

[source,java]
----
package io.microprofile.tutorial.store.product.resource;

import io.microprofile.tutorial.store.product.entity.Product;
import io.microprofile.tutorial.store.product.service.ProductService;
import jakarta.inject.Inject;
import jakarta.ws.rs.*;
import jakarta.ws.rs.core.MediaType;
import jakarta.ws.rs.core.Response;

@Path("/products")
@Produces(MediaType.APPLICATION_JSON)
public class ProductResource {

    @Inject
    private ProductService productService;

    @GET
    public Response getAll() {
        return Response.ok(productService.getProducts()).build();
    }

    @GET
    @Path("/{id}")
    public Response getById(@PathParam("id") Long id) {
        return productService.getProductById(id)
                .map(product -> Response.ok(product).build())
                .orElse(Response.status(Response.Status.NOT_FOUND).build());
    }
}
----

[cols="2,1,3"]
|===
|Endpoint |Method |Description

|`/catalog/api/products`
|GET
|Retrieves the full list of products as JSON

|`/catalog/api/products/{id}`
|GET
|Retrieves a single product by its ID
|===

The REST endpoints produce `application/json`, and the `@Path` annotation defines the URI structure for each resource.

In later chapters, this API will be extended with create, update, and delete operations (`POST`, `PUT`, `DELETE`) to implement full CRUD functionality.

== Running the Application

=== Using Liberty Maven Plugin

[source,bash]
----
mvn liberty:dev
----

This starts Open Liberty server in development mode with hot reloading enabled, allowing for rapid development cycles.

=== Accessing the Application

Once the server is running, you can access:

* Products Endpoint: http://<hostname>:5050/catalog/api/products

Replace <hostname> with _localhost_ or the hostname of the system, where you are running this code.

=== Running Unit Tests

To run the `ProductResourceTest.java` unit test, use the following Maven command from the project root:

[source,bash]
----
mvn test -Dtest=io.microprofile.tutorial.store.product.resource.ProductResourceTest
----

This will execute the test and display the results in the terminal.
